Train a topic model

Creating a sparse matrix

lyrics_sparse <-
  tidy_lyrics %>%
  count(song_name, word) %>%
  cast_sparse(song_name, word, n)

Adding predictors

What I first found confusing is that she first extracts the topic and THEN adds predictors as I thought that adding predictors in the first step may support the extraction of meaningful topics (as it is possible when using the stm() function as shown in the vignette by Roberts et al. 2019). Estimating the topic model along a range of topics


Instead of running the model with a fixed number of topics, running it across a range and inspecting some goodness of fit criteria may be the better option. In this case, I adapted Julia’s hacker corpus example (https://juliasilge.com/blog/evaluating-stm/) to the spice girls example.

library(furrr)
## Lade nötiges Paket: future
plan(multicore)

Note. In the hacker corpus example, Julia used plan(multisession).


many_models <- tibble(K = c(3, 4, 6, 7, 8)) %>%
  mutate(topic_model = future_map(
    K, ~stm(lyrics_sparse, K = ., verbose = FALSE), .options = furrr_options(seed = TRUE))
  )

Note. In Julia’s original example, she did not use the .options argument which resulted in a warning. Using this argument, however, is recommended as in this case, the algorithm generates L’Ecuyer-CMRG RNG seed random seed numbers (whatever this means ;)

Be that as it may, the central issue is that the above code generates a bunch of topic models (with the predescribed K’s) and stacks them into a nested tibble:

Creating evaluation criteria

In the next step, evaluation criteria are applied to (“mapped over”) the rows in the nested tibble:

heldout <- make.heldout(lyrics_sparse)

k_result <- many_models %>%
  mutate(exclusivity        = map(topic_model, exclusivity),
         semantic_coherence = map(topic_model, semanticCoherence, lyrics_sparse),
         eval_heldout       = map(topic_model, eval.heldout, heldout$missing),
         residual           = map(topic_model, checkResiduals, lyrics_sparse),
         bound              = map_dbl(topic_model, function(x) max(x$convergence$bound)),
         lfact              = map_dbl(topic_model, function(x) lfactorial(x$settings$dim$K)),
         lbound             = bound + lfact,
         iterations         = map_dbl(topic_model, function(x) length(x$convergence$bound)))



Plotting the four evaluation criteria


k_result %>%
  transmute(K,
            `Lower bound`         = lbound,

            `Held-out likelihood` = map_dbl(eval_heldout, "expected.heldout")) %>%

Topic correlations

As topic models are mixture models and each topic has a “value” (=gamma) on all topics, the topics may be unrelated or correlated. A positive correlation may indicate that documents having content about topic A tend also to present some content of topic B and a negative correlation may indicate that documents presenting topic A sytematically not represent topic B. The more the topic are excusive/non-overlapping, the more the correlations will strive towards -1.

In a video by Oskar Enghoff (https://www.youtube.com/watch?v=IC5BWQrCNUM&ab_channel=methodsMcr), he showed at 34:38 how to generate a topic correlation matrix and a network of correlating topics.

KM <- stm::topicCorr(topic_model)$cor %>% 
  as_tibble() %>% 
  mutate(across(everything(), round, 2)) %>% 
  as.matrix()
